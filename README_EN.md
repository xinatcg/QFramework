## Architecture Rule

**QFramework System Design Architecture has 4 layersÔºö**

* Presentation LayerÔºöViewController Layer. Using IController interfaceÔºåreceive input from user and state changed event from model. In unity MonoBehaviour is on presentation layer
    * Can get System
    * Can get Model
    * Can send Command
    * Can listen Event
* System LayerÔºöUsing ISystem interface. share IController's part of responsibility. Sharing logic shared across multiple presentation layersÔºåsuch-as time system„ÄÅshop system„ÄÅachievement system.
    * Can get System
    * Can get Model
    * Can listen Event
    * Can send Event
* Model LayerÔºöUsing IModel interface.Responsible for data definition, data addition, deletion, query and modification methods.
    * Can get Utility
    * Can send Event
* Utility LayerÔºöUsing IUtility interface.Responsible for providing infrastructure, such as storage method, serialization method, network connection method, Bluetooth method, SDK, framework inheritance, etc. Nothing can be done. You can integrate third-party libraries or encapsulate APIs
* In addition to the four layers, there is a core concept - command
    * Can get System
    * Can get Model
    * Can send Event
    * Can send Command
* Layer RuleÔºö
    * IController change ISystem„ÄÅIModel's state by Command
    * Notify iController after the change of ISystem and IModel must use event or BindableProperty
    * IController can get ISystem„ÄÅIModel for data query
    * ICommand cannot have state
    * The upper layer can directly obtain the lower layer, and the lower layer cannot obtain the upper object
    * Events for lower layer to upper layer communication
    * The communication between the upper layer and the lower layer is called by method (only for query and command for state change). The interaction logic of IController is special, and command can only be used
